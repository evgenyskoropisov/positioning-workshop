# Product Principles

These principles guide the product and every skill revision.

## 1. Evidence beats opinion

The plugin should start from source material, market signals, and proof points before asking the user to wordsmith anything.

## 2. Tradeoffs create clarity

The product should force hard choices around buyer, trigger, wedge, category, and proof. Good positioning excludes.

## 3. Proof and ambition must stay separate

The workflow should clearly distinguish what is true today from what the team hopes will be true later.

## 4. Substitutes matter

Real alternatives include manual work, internal teams, adjacent tools, and no-decision. The product should model those explicitly.

## 5. Team disagreement is useful

The product should surface disagreement instead of smoothing it over. Contradictions reveal strategic decisions that still need to be made.

## 6. Outputs must be reusable

The result should not stop at one positioning line. The plugin should produce assets that help with homepage messaging, sales conversations, and investor framing.
