# Positioning Variants

This skill adapts a chosen position across segments, channels, and contexts without breaking the core narrative.

Use it only after the team has already agreed on a core position.

## Core rule

The core spine stays fixed:

- buyer priority
- urgent trigger
- primary wedge
- proof standard
- category choice
- explicit boundary

Variants may change emphasis, examples, proof ordering, and CTA angle. They should not silently rewrite the strategy.

## Deliverables

Produce:

- a variant matrix
- recommended versions by segment or channel
- notes on where message drift begins
- updated copy snippets if requested

## Workflow

### Stage 1: Lock the spine

Ask for or extract the approved core position:

- one-line market position
- category sentence
- primary wedge
- strongest proof
- who it is not for

If the spine is still fuzzy, stop and tighten it first.

### Stage 2: Choose the adaptation axis

Pick one axis at a time:

1. segment
2. persona
3. company size
4. use case
5. acquisition channel
6. sales stage

Do not adapt across multiple axes at once unless the user explicitly asks for a matrix.

### Stage 3: Define invariants

List what cannot change:

- market position
- category choice
- wedge
- hard proof boundary

Then list what can flex:

- wording
- examples
- proof emphasis
- CTA framing
- ordering of ideas

### Stage 4: Build the matrix

Use `references/variant-matrix-template.md`.

For each requested variant, produce:

- who it targets
- what stays constant
- what changes
- which proof gets emphasized
- the CTA angle
- the drift risk

### Stage 5: Drift check

Reject or flag a variant if it:

- changes the market category
- changes the primary buyer without explanation
- introduces proof the company does not have
- turns a supported claim into a proven one
- contradicts the boundary of what the product is not

### Stage 6: Variant pack

Deliver the final set in the format the user needs:

- homepage
- segment page
- outbound email opener
- sales talk track
- investor short description

Keep the center of gravity the same across all of them.

## Writing rules

- Keep the core idea recognizable in every version.
- Change emphasis before changing meaning.
- Stay concrete and buyer-readable.
- Mark any new stretch as a bet.
